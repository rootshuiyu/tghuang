# 咪咕系统跑通步骤

## 1. 执行数据库脚本

在项目根目录下执行（或使用 phpMyAdmin / 其他客户端执行）：

```bash
# 确认数据库表前缀（.env 或 application/database.php 中的 prefix，如 fa_ 或 hub_）
# 若前缀不是 fa_，请先替换 data/migu_menu.sql 中所有 fa_ 为你的前缀
mysql -u你的用户 -p 你的数据库 < data/migu_menu.sql
```

或手动执行 `data/migu_menu.sql` 中的全部 SQL。

## 2. 清空缓存

- 方式一：后台右上角点击「清空缓存」
- 方式二：删除或清空目录 `runtime/cache`、`runtime/temp` 下内容

## 3. 刷新后台

重新登录或刷新后台首页，应看到：

- 顶部标题为「咪咕系统」
- 左侧/顶部菜单出现「游戏渠道列表」「游戏账号列表」「支付订单数据」「收银台地址」「抽佣点位记录」「分后台管理」等
- 右上角有「新订单通知」，有待支付订单时会显示数字角标，点击进入订单列表

## 4. 功能自测

| 功能 | 入口 | 说明 |
|------|------|------|
| 收银台地址 | 菜单「收银台地址」 | 列出当前管理员可见支付通道及收银台链接，可复制 |
| 抽佣点位记录 | 菜单「抽佣点位记录」 | 仅显示系统自动扣佣记录（支出+系统变动） |
| 新订单通知 | 顶部「新订单通知」 | 每 60 秒轮询近 24 小时待支付订单数，>0 显示红点，点击进订单列表 |

## 5. 若新订单通知不显示数字

- 打开浏览器控制台看是否有请求 `yunos/order/recent_count` 报错（如 404、403）
- 若 404：确认后台入口为 `index.php/admin/` 或 `admin.php`，且该路径下可访问 `yunos/order/recent_count`
- 若 403：确认已登录且该账号有订单权限（或 Order 控制器已设置 `noNeedRight = ['recent_count']`，已设置则无需权限）

## 6. 若菜单未出现「收银台地址」「抽佣点位记录」

- 在「权限管理」->「菜单规则」中查看是否存在 `yunos/cashier`、`yunos/commission`
- 若不存在，说明 INSERT 未执行成功（例如前缀不对或 `yunos/access` 不存在），请检查 SQL 执行报错并修正前缀后重跑
- 若存在但未显示，在「权限管理」->「角色组」中为当前角色勾选这两个规则并保存

## 7. 业务逻辑向咪咕靠齐

- 完整对齐清单与后续可做项见 **data/MIGU_ALIGNMENT.md**
- 执行 migu_menu.sql 后，除上述菜单外，资金流水、卡密、店铺、安全令牌等若存在也会被更新为咪咕文案
