API 对接文档

欢迎使用我们的支付API服务。本文档将帮助您快速集成我们的系统对接，实现高效、安全的数据业务支持。

全局说明 下单接口 查单接口 订单回调

全局说明

接口信息

接口名称 值
API网关 咨询业务获取
下单接口 /api/pay/bulid
查单接口 /api/pay/query

签名规则

签名类型 签名字符串
下单签名 商户appid + 商户appkey + 订单ID + 订单金额 + 时间戳
回调签名 商户appid + 商户appkey + 订单ID + 时间戳
查单签名 商户appid + 商户appkey + 订单ID + 时间戳

签名步骤

1

将需要签名的项目按照上方格式拼接好

2

将拼接好的字符串使用 SHA-256 生成即可，忽略 "+" 号

签名示例

以下是一个下单签名的完整示例：

  1. 步骤1：准备参数

    • 商户appid: 123456
    • 商户appkey: abcdef
    • 订单ID: ORD7890
    • 订单金额: 100 (元)
    • 时间戳: 1735581624

  2. 步骤2：按格式拼接字符串

    字符串
    123456abcdefORD78901001735581624

  3. 步骤3：使用SHA-256加密

    SHA-256
    8a7f5b3d1e9c6f2a4b0d8e7c3f5a9b0c2d4e6f8a1b3d5e7f9c1a3b5d7e9f2a4c
⚠️

金额如果为整数，请不要保留小数，例如：10元，使用10签名，而不是 10.00

API 下单接口 GET

接口地址

/api/pay/bulid

参数说明

参数 类型 必填 描述
appid string 是 商户appid
suporder string 是 商户订单号
fee number 是 订单金额
sign string 是 查单签名，签名方法看全局说明
timestamp number|max 10 是 时间戳
pay_type number 否 支付方式，微信支付:wxpay，支付宝:alipay
code string 否 通道代码
mid string 否 账号MID
return_url string 否 支付完成跳转链接
notify_url string 否 异步回调链接

下单接口-成功响应示例

JSON
{ "code":1, "msg":"ok", "time":1735581624, "success":true, "data":{ //平台订单号 "orderid" : "KM1450d8b8a7204668adde1e6cd404126b", //支付地址 "url":"http://134.122.169.7/api/pay/payment?orderid=KM1450d8b8a7204668adde1e6cd404126b" } }

下单接口-失败响应示例

JSON
{ "code":0, "msg":"SUP暂无可用账号", "time":1735581624, "success":true, "data":[] }

API 查单接口 GET

接口地址

/api/pay/query

参数说明

参数 类型 必填 描述
appid string 是 商户appid
suporder string 是 商户订单号
sign string 是 签名
timestamp number|max 10 是 时间戳

查单接口-响应示例

JSON
{ "code":1, "msg":"ok", "time":1735586637, "success":true, "data":{ "orderid":"KM948b2f5fbe074133acce41ca1262a997",//平台订单号 "suporder":"OR173558588746544",//商户订单号 "fee":"1.00", //订单金额 "status":10 //订单状态 0=正在取码，1=取码失败，2=等待支付，3=支付超时，10=支付成功 "exptime":"1735558680", //订单超时时间 } }

API 订单回调 POST

回调说明

接收到回调消息后，请返回 OK，只有成功支付的订单会回调。

⚠️

请确保您的回调接口能够正确处理重复回调的情况，建议通过订单号进行幂等处理。

订单回调-示例

Form-Data
{ "orderid": "KM7dfcdeac4bfe4c20b2e0a578b622cdda", "suporder": "OR173558771289925", "appid": "42783196", "fee": "1.00", "status": "10", "timestamp": "1735587730", "sign": "384352844212f5b9df47b07f88f1366ad2a2c54f13251bd4a85d01a695ee736d" }
